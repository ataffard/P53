Conditional Expressions

Relational expressions evaluate to either true (1) or false (0).
Be especially careful not to use a single equals (=), which is used for assigning a value to a variable, when you really want to test equality (==). 
x > 0 x < 0 x >= 0 x <= 0 x == 0 x != 0
Relational operators cannot be chained together, so you cannot write -1 < x < +1. Instead, use the logical operators described below.

Logical operators combine true/false values where anything equal to zero is considered false and any other numerical value is considered true. 
(...) && (...) /* true if both sides are true */ 
(...) || (...) /* true if at least one side is true */
You can chain together multiple logical operators into a single expression.
If you mix && and ||, remember that && has higher precedence (&& and || are analogous to * and +) 
(...) && (...) && (...) 
((...) || (...)) && (...)
The sub-expressions within a conditional expression are evaluated in a predictable and well defined order (generally left to right).
This is important in case any expressions have side effects (for example, you might call a function that updates a log file).
In case the final result can be deduced without evaluating all sub-expressions, the remaining sub-expressions are guaranteed not to be evaluated.
This is called "short-circuit evaluation" and allows you to write code such as the following. 
if(x >= 0 && sqrt(x) < 1) { /* sqrt(x) will never be called with x < 0 */ }
